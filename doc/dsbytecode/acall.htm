ACALL(dsByteCode::ebcACall,71) - Abstract Call Function
Description
Calls the given function abstract popping the needed amount of arguments off the argument stack, building an argument list from them and passing the execution over to the abstract fuction. Any function owned by one of the classes in the engine space can be used. Before execution the engine determines the most significent derivation of the abstract function in the execution object. Thus ACALL calls not the original function but the best derivation findable in the execution object. If you do not intend this use CALL instead. The function to call has to be abstract and not static. The execution object is taken as the object the function is called upon. A non-static function can not be called using ACALL (static functions can not be abstract).
Syntax
[byte]        dsByteCode::ebcACall
[dsFunction*] function
Parameters
function
Function to call. This function has to be member of one of the classes existing in the engine space. Arguments stored for calling the function have to be stored in reverse order (argument is a stack). The function expects them to be in correct order upon execution.
