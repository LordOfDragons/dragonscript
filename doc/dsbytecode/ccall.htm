CCALL(dsByteCode::ebcCCall,72) - Constructor Call Function
Description
Calls the given constructor function popping the needed amount of arguments off the argument stack, building an argument list from them and passing the execution over to the fuction. Any constructor function owned by one of the classes in the engine space can be used. Before the constructor function is invoked the engine creates a new object of the type matching the function's owner class. Thus CCALL has a similar effect as the commonly known new keyword found in C++ language. The newly created object will be placed in the execution object and the function is called upon this.
Syntax
[byte]        dsByteCode::ebcCCall
[dsFunction*] function
Parameters
function
Constructor function to call. This function has to be member of one of the classes existing in the engine space and has to me declared as beeing a Constructor. Arguments stored for calling the function have to be stored in reverse order (argument is a stack). The function expects them to be in correct order upon execution.
