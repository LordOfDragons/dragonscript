CSTR(dsByteCode::ebcCStr,4) - Const String Value
Description
Creates a new constant string and stores it as the current execution object. At compile time any string literal encountered will be inserted into the engine string table. Each string in the table is refered to by it's index position. Together with the CSTR command the index of the corresponding string in the table is stored. Using this index the engine core creates a result from the string table entry each time this command is encountered.
Syntax
[byte] dsByteCode::ebcCStr
[int]  index
Parameters
index
Integer index of the string in the constant string table of the script engine. to retrieve this value take the string you want to load and mangle it through
int dsEngine::AddConstString(const char *String)
with the index of the string beeing the return value of this function.
