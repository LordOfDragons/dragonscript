DragonScript - ByteCode
Introduction
DragonScript Syntax uses bytecode internally to execute the scripts. The compiler creates bytecode from the scripts alike that the script engine can execute it directly. This bytecode is stored in objects of type dsByteCode and only the script engine classes should modify them. If for some reason you definitly need to write bytecode yourself you can use this guide to find out how. Please keep in mind that this bytecode is the heart of the script engine and that the smallest error can crash your application. The script engine does not carry out checks on all data which usually are not modified outside to gain speed. Thus make sure your bytecode is 100% correct at all times.
