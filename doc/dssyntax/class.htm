Class Definition
Description
A Class is the basic unit of every script you write. It's a definition of the objects you can create using this class as it's type. Classes are refered to throughout this documentation as types. The class definition consists of two different pieces:
  • The class header: containing the name of the class and inheritance information
  • The class body: containing the definition of all class members
Syntax
class class-name
[ extends base-class ]
[ implements interface1 [ ',' interface1 ... ] ]
'newline'
   [ inner class definition ]
   [ inner interface definition ]
   [ enumeration declarations ]
   [ variable declarations ]
   [ function declarations ]
'end'
Parameters
class-name
The class-name must be unique in the namespace the class definition appears in. If you think you might get into trouble using a certain class name you can enclose the class definition in a namespace definition to make it unique again.
base-class
Each class is derived from another class to build a hierarchy of classes. If you do not specify a base class here the parser will use Object as the base class. You can specify any class defined in either the scripts already loaded or any scripts the given package contains where this script belongs to. A class must have only one base class.
interface1, interface2, ...
A class can have one or more interfaces. The class inherits from each interface the functions and has to implement them. A class definition is only valid if all functions defined in the interfaces listed here have been implemented. If interfaces have duplicate functions an error will be issued.
inner class/interface definitions
A class can have inner classes and interfaces. You can then access those inner classes/interfaces with the usage of the '.'-operator.
Example
Simple class definition.
class MyScript
   // member declarations
end
A more elaborate example of a class definition including base class and interfaces 
class MyScript extends BaseScript implements MouseEvent, RenderEvent
   // member declarations
end
Classes can also contain nested inner classes. This way you can create a hierarchy of classes. You can then refer to this class using the '.'-Operator like MyClass.InnerClass. 
class MyClass
   class InnerClass
      /* member declarations */
   end
   // member declarations
end
