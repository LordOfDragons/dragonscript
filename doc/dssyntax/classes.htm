Classes and Interfaces
Description
Classes are the heart of any script written in DragonScript. A class encapsulates the information needed by the script engine to create objects and maintain its data. A Class definition includes the name of the class, heritage informations like the base class and optional interfaces and all the class members that are accessible in objects created from this class. Thus a class is like a template defining how object look and behave like.
