Class Enumeration Declaration
Description
Enumerations are a short way for declaring constant variables or short constants. Each enumeration entry creates a constant variable of type int having the value of the previous enumeration entry plus 1 or 0 if it is the first entry in the list. You can also overwrite this behaviour and specify your own value for certain or all entries.
Syntax
enum enum-name
'{'
   constant1 [ '=' value ] ',' ...
'}'
Parameters
enum-name
Is the identifier name of the enumeration. Up to now this is merly a informative value having no influence on the script itself. It can be used by editors to group enumeration constants.
constant1
The name of one constant in the enumeration. This name has to be unique in the class this enumeration appears in.
value
If you specify no value the constant will be assigned the value of the precedor entry plus 1 or 0 if it is the first entry in the list. The value has to be a constant integral value meaning it has to be an int value.
Example
Simple enumeration declaring a few colors. 
class MyScript{
   enum eColors{
      ecRed, // will be 0
      ecGreen, // will be 1
      ecBlue // will be 2
   }
   // member declarations
}
This time we change the values of the constants. 
class MyScript{
   enum eColors{
      ecRed, // will be 0
      ecGreen, // will be 1
      ecBlue, // will be 2
      ecBlack = 10, // will be 10
      ecWhite // will be 11
   }
   // member declarations
}
