Requires Statement
Description
If your script needs certain other script packages, files or classes that need to be loaded before the script can be parsed you use the requires statement to load those packages before the parsing of the script takes place. The statement takes a string identifying the package to load.
Syntax
requires "package-name" ';'
Parameters
package-name
A string specifying the name of the package to load before parsing the script. All packages mentioned this way no matter where the statement resides in the script will be loaded before anything gets parsed. What string you have to specify there depends on the implementation of the host application. Usually this string indicates the name of a script package to load. Please refer to the documantion of that application to learn about howe this string is interpreted.
Example
Most applications and the compiler use package names as the way to identify needed script packages. This example will load the OpenGL package if the host application provides those.
requires "OpenGL";
Certain applications support loading packages from different sources. In this example we want to load a package from the internet to provide OpenGL functionalities. That kind of source independent package management can come in handy for scripts not bound to a certain PC. 
requires "http://my.domain.ch/packages/OpenGl";
Like states above the possible ways of including a package depends on the host application.
