Select Statement
Description
The Select Statement executes statements depending on the value of a mode-expression. You can then specify a set of Cases for this mode-expression. On execution the value in the mode-expression is matched against all available Cases and in the case of a match the apropriate statement is executed. In all other cases the default statement is executed or all Cases skipped if such a statement does not exist.
Syntax
select mode-expression {
   case value1 [ , value2 ... ] :
      statement-block
   [ ... ]
   [ else
      else-statement-block ]
}
Parameters
mode-expression
Is an expression of type or castable to int. This value will be matched against all Cases.
value1, value2, ...
Defines the values the mode-expression has to be for the Case to apply. The values have to be unique in the Select Statement they are defined and have to be constant integer values.
statement-block
The statement to execute if the Case matches the mode-expression. After executing the statement the other Cases including the default case are skipped.
else-statement-block
If no Case matches this statement is executed. If the statement is missing all Case statements are skipped.
Example
Select-Statements are usefull if there is a case where you want to execute code depending on the value of an expression. You could use lots of cascadet If-Else-Statements to do so or you can simply use a Select-Statement. 
select vMode{
   case 0: // executes if vMode is 0
      vResult = vOp1;
   case 1, 2, 3: // executes if vMode is 1, 2 or 3
      vResult = vOp2;
   case CONST_VALUE: // executes if vMode equals the
      // constant class variable CONST_VALUE or the
      // enumeration variable CONST_VALUE
      vResult = vOp1 + vOp2;
   else // in all other cases do this
      vResult = 0;
}
